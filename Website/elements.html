AWS Cloud Resume Challenge

As part of my commitment to continuous learning in cloud computing, I completed the Cloud Resume Challenge created by Mr. Forrest. This project involved building a cloud-based resume using various AWS services, getting my hands dirty with AWS cloud services and infrastructure as code. The project involved not only showcasing my skills but also implementing best practices in cloud security and scalability. By leveraging a combination of AWS tools, I was able to create a seamless user experience while ensuring that my resume is easily accessible and maintainable. The challenge also emphasized the importance of automation and efficient deployments, which I achieved using Terraform for infrastructure management and thanks to this challenge I was able to build this awesome portfolio site implementing everything I have previously learnt.

Key Features:

  • Hosted a static portfolio website on AWS S3 with a custom domain.
  • Implemented a visitors counter feature using AWS Lambda and DynamoDB.
  • Set up a CI/CD pipeline for automated deployments using Terraform.
You can explore the complete implementation and source code in my GitHub repository: AWS Cloud Resume Challenge

AWS Three-Tier Web Application

  • Architected and deployed a highly available three-tier application architecture using Amazon EC2, RDS, VPC, subnets, security groups, and Route 53.
  • Designed network segmentation and access controls aligned with AWS security and availability best practices.

Cloud Launch Project

  • Deployed a secure static website using Amazon S3 and CloudFront with restricted IAM access.
  • Designed a VPC network layout with public and private subnets, improving isolation and security.
  • GitHub: Cloud Launch Project

Dynamic Prototype Web Application

  • Deployed and supported a dynamic web application on Amazon EC2 (Ubuntu 22.04 LTS) using Nginx.
  • Configured SSL/TLS certificates using Let's Encrypt (Certbot) and managed DNS records.
  • Integrated Amazon CloudFront for secure, globally distributed content delivery.
  • GitHub: Dynamic Prototype Web Application Project
